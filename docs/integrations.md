# Social Platform Integration Guide

CreatorFlow ships with a modular integration layer inside `backend/app/services/social` that keeps each
platform adapter isolated behind a common publishing interface. This document explains how to
connect the sandbox stubs that ship with the repository to production APIs for every supported
network.

## Common Concepts
- **OAuth storage** â€“ OAuth and long-lived access tokens are persisted in MongoDB's `oauth_tokens`
  collection via `services/social/store.py`. Tokens are encrypted with `utils/crypto.py` before they
  are written to the database. Update the encryption secret through the `APP_SECRET_KEY` environment
  variable.
- **Queue based publishing** â€“ Every publish request flows through `queue.py` and is processed by the
  `worker` service defined in `docker-compose.yml`. Time-sensitive actions (story expirations,
  scheduled posts, etc.) should enqueue a job with metadata describing the platform and
  post payload. The worker dispatches to the correct adapter.
- **Adapter contract** â€“ Each adapter exposes `authorize()`, `refresh_token()`, `publish()` and
  optional `fetch_analytics()` helpers. New platforms can plug into the dispatcher by adding their
  adapter to `services/social/registry.py`.

## Instagram (Graph API)
1. Create a Facebook Developer App and enable the Instagram Basic Display and Graph API products.
2. Populate the following environment variables in `backend/.env`:
   - `INSTAGRAM_CLIENT_ID`
   - `INSTAGRAM_CLIENT_SECRET`
   - `INSTAGRAM_REDIRECT_URI`
3. Implement `InstagramAdapter` by exchanging the authorization code inside
   `authorize()` and refreshing the long-lived token in `refresh_token()`.
4. For publishing, supply media via the `/media` edge, poll the creation status, and publish using the
   `/media_publish` endpoint. `publish()` already receives normalized assets generated through the
   media pipeline (`services/media`). Map the values to the Graph API payload.
5. Schedule monitoring jobs to rotate tokens every 50 days and to respect rate limits. Use the Redis
   queue to space requests when uploading batches.

## TikTok
1. Create a TikTok for Developers application and retrieve the client ID/secret, then define
   `TIKTOK_CLIENT_ID`, `TIKTOK_CLIENT_SECRET`, and `TIKTOK_REDIRECT_URI` in `backend/.env`.
2. The adapter should implement the OAuth 2.0 PKCE flow. Store `code_verifier` values in Redis during
   the handshake to avoid replay attacks.
3. Upload videos with the [TikTok Upload API](https://developers.tiktok.com/doc/content-posting-api-overview/):
   - Initialize the upload to receive an upload URL.
   - Transfer binary data from the pre-rendered MP4 generated by FFmpeg.
   - Confirm the upload and set captions/cover frames.
4. TikTok requires content review before publishing. Track the task ID returned by the confirmation
   API and poll using `getVideoList` to confirm success before marking jobs as complete in MongoDB.

## YouTube
1. Create OAuth credentials (type "Web Application") and set `YOUTUBE_CLIENT_ID`,
   `YOUTUBE_CLIENT_SECRET`, and `YOUTUBE_REDIRECT_URI`.
2. Because YouTube access tokens expire in one hour, make sure `refresh_token()` handles refresh
   tokens and updates the encrypted payload in MongoDB.
3. For uploads, leverage the Resumable Upload flow (`googleapiclient.discovery.build`). The
   CreatorFlow worker converts the video to an MP4 with the correct aspect ratio; the adapter only
   needs to relay metadata (title, description, tags, privacy status).
4. After publishing, call the Analytics API to populate `fetch_analytics()` with views, watch time, and
   CTR. Cache results in Redis (`services/optimization/cache.py`) to keep dashboards responsive.

## LinkedIn
1. Register an app in the LinkedIn developer portal and add your redirect URI.
2. Set `LINKEDIN_CLIENT_ID` and `LINKEDIN_CLIENT_SECRET`. LinkedIn requires the "w_member_social" and
   "rw_organization_admin" scopes for user and company page posting.
3. The adapter should call `/ugcPosts` for rich media updates and `/shares` for text-only posts.
   Provide organization URNs during scheduling to differentiate personal vs. company publishing.
4. Fetch analytics using `/organizationalEntityShareStatistics` and backfill the `analytics` collection
   for comparative reporting.

## Twitter (X)
1. Enable Elevated or Enterprise access to the Twitter API v2 and configure `TWITTER_CLIENT_ID`,
   `TWITTER_CLIENT_SECRET`, and `TWITTER_REDIRECT_URI`.
2. CreatorFlow prepares text variants with character counts; the adapter validates the payload before
   sending it to `/2/tweets`. For threads, iterate over the `segments` field provided by the
   optimization layer.
3. Pull engagement metrics using the `/2/tweets/:id` endpoint with the `tweet.fields=public_metrics`
   query parameter. Store results in the analytics pipeline for predictive modeling.

## Facebook
1. Use the same Facebook Developer app created for Instagram and add `FACEBOOK_CLIENT_ID` and
   `FACEBOOK_CLIENT_SECRET` to the environment.
2. Publish content through the `/feed` or `/photos` edge depending on the media type. The adapter
   receives brand assets, copy, and scheduling metadata to map to the Graph API.
3. Respect per-page rate limits by batching jobs in Redis and retrying with exponential backoff using
   the helper in `utils/retry.py`.

## Pinterest
1. Register a Pinterest developer application and capture credentials in
   `PINTEREST_CLIENT_ID`, `PINTEREST_CLIENT_SECRET`, and `PINTEREST_REDIRECT_URI`.
2. CreatorFlow stores pre-rendered portrait images in S3 or local object storage; the adapter uploads
   the media via the `/v5/pins` endpoint and attaches board IDs derived from workspace settings.
3. Fetch `analytics/metrics` to expose saves, impressions, and outbound clicks inside the analytics
   dashboard.

## Emerging Platforms
1. Create a new adapter class under `services/social` implementing the shared interface.
2. Add the adapter to `registry.py` and expose configuration flags in `config.py`.
3. Update the frontend integrations page to display connection settings by inserting a new record into
   `stores/useIntegrationsStore.ts`.

## Webhooks and Rate Limits
- Queue webhook callbacks with Redis to avoid request storms.
- Store delivery history inside MongoDB (`webhook_logs`) for auditing and replay.
- Use the `utils/rate_limit.py` helper to throttle outbound calls per platform.

## Testing
- Mock every platform client inside the `tests/services/social` suite.
- Record VCR cassettes for integration tests to guarantee deterministic builds.
- Validate adapters end-to-end in a staging environment before enabling them for production
  workspaces.
